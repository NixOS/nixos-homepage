Documentation is infrastructure. The NixOS Documentation Team keeps it reliable.
We own the standards, tooling, and review processes that keep NixOS documentation consistent across the entire ecosystem - NixOS, the Nix package manager, nixpkgs, and the Nix language itself.
We review contributions, maintain the writing guides contributors follow, and own the infrastructure pipelines.
We guide community efforts to keep documentation accurate, reliable, and accessible.

## Vision

Our goal is a unified documentation portal that clearly separates reference material from practical guides, organized by ecosystem for easy navigation.
We consolidate scattered resources, eliminate duplication, and establish a single source of truth for the community.
We distinguish between curated documentation and the community wiki, allowing each to serve its distinct purpose.
This lowers barriers for newcomers while preserving the depth experienced users rely on.

[Read the detailed vision](https://github.com/NixOS/nix.dev/tree/master/maintainers/vision.md)
